1. Reading files correctly

From the machine perspective, a file is a sequence of bytes stored on disk. If the file contains textual data, then each character in the file is encoded into one or more bytes using some charset encoding. ASCII, UTF-8, and UTF-16 are examples of character encodings. My machine uses UTF-8 by default. Reading text files one byte at a time (1 byte per disk IO) may or may not be correct nor efficient, depending on the charset encoding. How about an example. To read the bytes, we can use an InputStream, a superclass of all classes representing an input stream of bytes. Since we are reading a file, we are going to use the FileInputStream implementation.

We instantiated a FileInputStream inside a try-with-resources block to handle the closing of the stream automatically. Most I/O operations throw the checked IOException, thus we are obliged to catch this exception or include it in the throws clause of the enclosing method.

The int read() method, reads the next byte from the input stream and returns -1 when the stream is empty. In this example we are using a while loop to read every byte in the file, one byte at a time, casting it to its corresponding ASCII character and printing it to the standard.
